Other Tips

Listed here are some other tips to follow when writing documentation, in no particular order.

Note

This section is not complete.

  • Use example.com for URL examples, per RFC 2606.

  • Use a spell checker, especially if you have bad spelling habits!

  • Always test the documentation. See generating the manual for information on how.

  • Use hyphens (-) instead of underscores (_) for id attributes and entity names.

  • There are some entities that have been created for certain filenames. Use them instead of writing the whole filename out with <filename> tags

    • php.ini --> &php.ini;

    • .htaccess --> &htaccess; (notice, no dot)

    • config.php --> &config.php;
